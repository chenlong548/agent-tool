# Security Policy

## Supported Versions

Security fixes target the latest released version of Agent Tool.

## Reporting A Vulnerability

Please report security issues privately by opening a GitHub security advisory or contacting the repository owner.

Do not include secrets, private project files, or production credentials in reports.

## Security Notes

Agent Tool is a local workflow initializer and state machine. It does not require cloud credentials by default. Release workflows generated by the tool may ask users to reason about deployment, secrets, backups, rollback, and observability before production changes are made.
