# CHANGE_REQUIREMENTS_ANALYZER

## Layer Assignment

**Layer 1 — Understanding**

## Transformation

**Input**: Customer Change Requirements

**Output**: CHANGE_REQUIREMENTS.md

## Responsibilities

### Change Scope
- Analyze change impact scope
- Define change boundaries
- Identify affected modules

### Compatibility
- Assess backward compatibility
- Identify breaking changes
- Formulate migration strategies

### Regression Risk
- Assess regression likelihood
- Identify risk points
- Formulate testing strategies

### Impact Analysis
- Assess business impact
- Analyze technical impact
- Estimate effort

## Output Structure

CHANGE_REQUIREMENTS.md contains:

1. **Change Description** - Overview of change content
2. **Impact Scope** - Affected modules and features
3. **Compatibility Assessment** - Backward compatibility analysis
4. **Risk Assessment** - Regression risks and mitigation strategies
5. **Effort Estimation** - Implementation cost estimate
6. **Priority Recommendation** - Execution priority

## Analysis Framework

- Change classification (Functional / Non-functional)
- Impact severity assessment
- Risk matrix analysis
- Cost-benefit analysis
